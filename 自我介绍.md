# AlphaNexus 自我介绍

我做了一个多智能体股票投研系统 AlphaNexus，核心目标是把「信息收集-分析-辩论-风控-结论」做成可追溯的闭环，而不是让单一模型一次性给黑箱答案。

## 系统架构

基于 StateGraph 做多节点编排：先由市场、社媒、新闻、基本面四类分析员产出报告，再进入多空投资辩论，之后由交易员生成计划，再进入激进/中性/保守三方风控辩论，最后由风险裁判输出 Buy/Sell/Hold。整个过程通过 SSE 实时回传进度与辩论记录，用户可以看到每一步的依据。

## 产品增强

我在产品侧做了两个关键增强：

1. **「投资风格」模块**：支持五档风险偏好，并把偏好只注入新闻筛选层，避免影响辩论和最终决策的中立性。

2. **前端交互升级**：增加独立介绍页、可视化流程和市场实时预览，降低用户等待分析结果时的焦虑。

## 工程稳定性

在工程稳定性上，我做了多层容错：LLM key 的环境变量回退、数据源 vendor fallback、Alpha Vantage 多 key 轮换，以及组合服务的 MCP + 缓存降级机制，减少外部 API 限流导致的中断。

## 核心能力

我这个项目最想体现的能力是：**不仅会调用 LLM，而是能把多角色协作、决策透明、系统可靠性和用户体验放在一个完整产品里落地。**
