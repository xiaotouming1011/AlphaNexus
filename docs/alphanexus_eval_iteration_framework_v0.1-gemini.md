# AlphaNexus 多智能体投研系统评测与迭代框架 v0.1

## 1. 目标与范围

**目标**：规范 AlphaNexus 投研多智能体系统的产品质量基线与迭代标准，确保从信息处理到最终交易决策的全链路（分析 -> 多空辩论 -> 交易计划 -> 风控辩论 -> 决策）具备高可靠性、高逻辑连贯性与可解释性。

**范围**：
*   **多智能体协同评估**：多空辩论机制的交锋质量与风控环节的拦截有效性。
*   **底层生成质量**：幻觉控制、数据溯源准确性及指令遵循度。
*   **系统工程体验**：SSE 流式响应性能、基于 `risk_profile` 的个性化展示逻辑验证。
*   **评测运营机制**：从评测集建设、Badcase 闭环到发布门禁的完整 SOP。

---

## 2. 评估体系（主观维度 + 客观维度 + 评分权重）

### 2.1 客观维度 (Objective Metrics) - 占比 40%
基于自动化脚本进行规模化测试，主要关注工程规范与事实基准。

*   **来源追溯准确率 (Source Tracking Accuracy)**：生成的论点或数据能够精确匹配输入文档（市场/社媒/新闻/基本面）的比例。
    *   *KPI 建议值*：≥ 95%
*   **指令遵循度 (Instruction Following)**：
    *   严格按照用户 `risk_profile` 过滤/聚合新闻展示层。
    *   最终结论强制收敛于标准的枚举值（Buy / Sell / Hold）。
    *   *KPI 建议值*：100% (红线指标)
*   **流式响应性能 (Latency & SSE)**：
    *   首字响应时间 (TTFT, Time To First Token)。*KPI 建议值*：≤ 1.5s
    *   整体链路端到端延迟（多 Agent 并行/串行执行总耗时）。*KPI 建议值*：≤ 15s
*   **格式合规率 (Format Compliance)**：交易计划及内部 Agent 通信使用 JSON/Markdown 模板的解析成功率。
    *   *KPI 建议值*：≥ 99.5%

### 2.2 主观维度 (Subjective Metrics) - 占比 60%
由投研专家 (Expert) 介入评审，评估多智能体系统的“智商”与“财商”。采用 1-5 分制（5分为极佳）。

*   **逻辑交锋质量 (Debate Coherence) - 权重 25%**：多空辩论与风控辩论是否言之有物，是否具备深度博弈，严禁“和稀泥”或单方面压倒而缺乏反驳。
*   **信息提取广度与深度 (Info Extraction) - 权重 15%**：面对海量且嘈杂的社媒和新闻，核心情绪与基本面关键指标（如财报超预期、突发利空）是否被准确抓取无遗漏。
*   **风控合理性 (Risk Control Rationality) - 权重 10%**：风控 Agent 是否能针对交易计划中的漏洞（如止损点位设置不合理、仓位过重）提出有效的质询。
*   **最终决策说服力 (Decision Conviction) - 权重 10%**：最终得出的 Buy/Sell/Hold 结论是否与其前置的辩论和交易计划逻辑自洽。

## 3. 评测集建设与版本化

建立动态的评测数据集，按难度与用途分层，确保评测的覆盖面与科学性。

### 3.1 数据的获取与清洗
*   **获取来源**：历史研报金标准、真实市场极端波动事件截面数据、线上真实用户查询的 Ticker 与 `risk_profile` 组合。
*   **清洗规则**：脱敏用户隐私，剔除缺失多空博弈背景的“单边行情”平庸数据，补充带噪音的干扰新闻以增加鲁棒性测试。

### 3.2 评测集分层机制
*   **L1 冒烟测试集 (Smoke Test) - 50 条**：覆盖主流明星股 (如 AAPL, TSLA) 与基础风险偏好，验证系统主链路畅通与 SSE 正常推流。
*   **L2 回归测试集 (Regression) - 500 条**：覆盖各行业 Ticker、长尾标的，以及包含信息缺失、冲突等异常边界场景，用于日常迭代防劣化。
*   **L3 挑战测试集 (Challenge) - 100 条**：高难度测试集，包含历史上著名的“多空双杀”事件、极其嘈杂的社媒舆情，用于评估 Agent 模型升级或架构大改的认知上限。

### 3.3 版本化更新
*   **命名规范**：`AlphaNexus-Eval-v[YYYYMM]-[Level]` (例: `AlphaNexus-Eval-v202310-L2`)。
*   **更新频率**：每月汰换 15% 的 L2 测试用例，引入上个月的真实市场 Badcase 及新发酵的热点事件，保持评测集的新鲜度。

## 4. 线上 Badcase 运营闭环

构建“发现-归因-修复-回归”的标准闭环，驱动系统表现螺旋上升。

*   **1. 发现 (Discovery)**：
    *   通过前端埋点收集用户“踩 (Downvote)”与意见反馈。
    *   内部运营团队每周按 `risk_profile` 分层抽检 50 笔真实决策记录。
    *   自动化监控：抓取无来源引用的断言、Buy/Sell/Hold 解析失败的会话。
*   **2. 归因 (Attribution)**：针对 Badcase 标记责任层：
    *   *数据源层*：新闻/基本面 API 数据滞后或缺失。
    *   *信息提取层*：单 Agent 对超长文本产生了幻觉或关键遗漏。
    *   *协同决策层*：辩论过早妥协，或风控 Agent 被绕过。
    *   *展示层*：`risk_profile` 权重计算错误，导致新闻呈现不匹配。
*   **3. 修复 (Remediation)**：
    *   调整相关 Agent 的 System Prompt。
    *   优化 RAG 检索链路的 Chunking 策略或相似度阈值。
    *   调整多智能体 Workflow 的温度值 (Temperature) 或多轮辩论次数限制。
*   **4. 回归 (Regression)**：
    *   修复后的用例必须沉淀至 L2 评测集。
    *   **SLA 要求**：P0 级 Badcase (如给出确定性致损交易建议、严重幻觉) < 24h 修复；P1 级 < 3天。

## 5. QA 知识库体系

沉淀优质投研逻辑，作为 RAG 增强与 Agent Few-shot 学习的燃料。

*   **收集标准**：人工判定为 5 分的优秀多空辩论过程、资深分析师撰写的高质量风控策略标准范本。
*   **数据结构**：采用 `[Ticker] + [输入环境截面] + [Risk Profile] + [优秀辩论记录/Golden Response] + [交易计划]` 的 JSON 结构化存储。
*   **维护与应用**：
    *   建立基于 Embedding 的向量检索库。
    *   Agent 启动时，根据当前查询的 Ticker 和行业，动态检索 Top-K 相似的 QA 案例注入 Prompt 作为 Few-shot，提升输出稳定性。
*   **关键指标**：
    *   知识库检索命中率 (Hit Rate)：*KPI ≥ 80%*。
    *   加入 QA 动态注入前后的主观评分提升幅度：*目标提升 ≥ 10%*。

---

## 6. 发布门禁与迭代节奏

制定严格的代码与 Prompt 发布规范，保障线上服务的稳定性与投研专业度。

*   **周度发版 (Hotfix / Prompt 微调)**：
    *   *要求*：必须 100% 跑通 L1 冒烟测试集。
    *   *目标*：修复当周 Top 3 高频 Badcase，微调单 Agent 的输出语气或格式。
*   **月度发版 (功能新增 / Agent 逻辑更新)**：
    *   *要求*：跑通 L1 + L2 测试集。
    *   *门禁*：新版本在 L2 上的客观指标不低于基线，主观评分（抽样 20% 人工盲测）下降幅度 ≤ 2%。
*   **大版本发布 (底层模型升级 / 协同架构重构)**：
    *   *要求*：全量跑通 L1 + L2 + L3 测试集。
    *   *门禁*：L3 挑战集上的人工盲测胜率必须 ≥ 55%（显著优于旧版本）。
*   **绝对红线 (拦截发布)**：
    *   出现严重违背合规逻辑的输出。
    *   最终决策未输出明确的 Buy/Sell/Hold 比例 > 1%。
    *   TTFT > 3s (严重影响流式体验)。

## 7. v0.1 边界与后续演进

### 7.1 v0.1 版本边界
*   本版本主要聚焦于**链路连通性验证**、**基础评测集搭建**与**人工评测 SOP 的跑通**。
*   暂不引入基于大模型自动化裁判 (LLM-as-a-Judge) 的全量主观评分替代。
*   回测环节暂时依赖人工投研专家的经验评估，暂未接入历史实盘 K 线数据的自动化盈亏验证。

### 7.2 后续演进规划 (v0.2+)
*   **引入 LLM-as-a-Judge**：使用更高参数量级模型（如 Gemini 1.5 Pro）作为裁判 Agent，自动化批量评估“多空辩论质量”与“风控逻辑严密性”，降低人工标注成本。
*   **实盘回测对齐**：将 Agent 输出的交易计划（买入点、止损点）与真实历史行情 K 线打通，引入客观的“模拟盈亏比 (PnL)”作为核心评测指标。
*   **反思机制 (Reflection) 评测**：增加对 Agent 在发现逻辑漏洞后自我纠错能力的专项评估。
